# Codex 落地方式

Codex 适合处理本地项目里的工程化任务。

适合场景：

- 分析 CSV / Excel
- 扫描文件夹
- 生成 Markdown 文档
- 修改代码
- 整理 GitHub 仓库
- 生成知识库草稿
- 把重复操作脚本化

## 推荐结构

```text
User Goal
  -> Inspect files
  -> Produce plan
  -> Edit / generate artifacts
  -> Run verification
  -> Commit / deliver
```

## 示例：知识库文章批量检查

Codex 负责：

1. 读取本地导出的知识库文档。
2. 扫描禁用表达。
3. 检查文章长度和结构。
4. 识别缺少案例、步骤、图示的文章。
5. 生成整改清单。
6. 按优先级改写文章。

规则负责：

1. 不能删除用户已有内容。
2. 不能泄露内部资料。
3. 不能把销售策略写进用户可见文章。
4. 修改后必须复扫。

## 常见错误

1. 只在文章末尾追加内容，没有整体重构。
2. 生成了内部提示语。
3. 没有验证格式。
4. 没有沉淀可复用模板。

## Checklist

- [ ] 是否先扫描现状
- [ ] 是否识别高风险文档
- [ ] 是否保留原内容
- [ ] 是否整体重构而不是末尾追加
- [ ] 是否复扫输出
