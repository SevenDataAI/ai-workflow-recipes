# n8n 落地方式

n8n 更适合做自动化编排和系统集成。

适合放进 n8n 的部分：

- 定时触发
- Webhook 触发
- 表单接收
- 飞书 / 企业微信 / 邮件通知
- CRM / 表格 / 数据库写入
- 条件判断
- 审批流转

不建议完全放进 n8n 的部分：

- 长文本复杂推理
- 大规模代码修改
- 高复杂度 Agent 任务
- 需要大量上下文的文档重构

## 推荐结构

```text
Trigger
  -> Data Normalize
  -> AI Node
  -> Rule Check
  -> Human Approval
  -> External Action
  -> Log Result
```

## 示例：微信线索跟进

n8n 负责：

1. 接收线索消息。
2. 写入飞书多维表格。
3. 调用 AI 生成回复草稿。
4. 根据意图设置跟进时间。
5. 推送给人工确认。

AI 负责：

1. 判断用户意图。
2. 生成自然回复。
3. 总结用户关注点。

规则负责：

1. 没有明确购买意向前不发付款入口。
2. 用户没有说明背景时先追问。
3. 企业线索标记为高优先级。

## 常见错误

1. 没有记录用户来源。
2. 回复生成后直接发送，没有人工确认。
3. 没有跟进状态。
4. 把价格、链接、资料全部一次性发出去。

## Checklist

- [ ] 是否有 trigger
- [ ] 是否记录 source
- [ ] 是否有人工确认
- [ ] 是否有 follow-up 时间
- [ ] 是否记录最终结果
