# 视频选题到发布计划工作流

## 场景

你有一个选题，但每次都要重新想脚本、封面、标题、平台文案、置顶评论和私信回复。

这类工作有创意部分，也有固定流程。

创意部分可以交给 AI；固定流程应该固化成 workflow。

## 目标

输入一个选题，输出：

- 视频标题
- 3 个开头钩子
- 完整口播脚本
- 分镜建议
- 抖音文案
- B 站标题和简介
- 小红书文案
- 置顶评论
- 私信回复话术
- 发布检查清单

## 为什么不能只靠一次 Prompt

一次 Prompt 很容易出现三个问题：

1. 标题很猛，但脚本没有信息密度。
2. 脚本能看，但平台文案和评论引导断层。
3. 每次输出风格不稳定，没办法沉淀自己的内容资产。

所以更合理的做法是把它拆成多个节点。

## Workflow

```text
选题输入
  -> 受众判断
  -> 痛点拆解
  -> 核心观点提炼
  -> 脚本生成
  -> 分镜生成
  -> 平台文案生成
  -> 评论/私信引导
  -> 人工审稿
  -> 发布清单
```

## 哪些交给 AI

- 受众痛点拆解
- 钩子生成
- 脚本初稿
- 分镜建议
- 多平台文案改写

## 哪些必须规则化

- 不能夸大收益
- 不能承诺确定结果
- 必须保留评论区关键词
- 必须输出三个平台版本
- 必须生成私信承接话术
- 结尾必须引导一个明确动作

## 人工确认点

- 案例是否真实
- 是否泄露内部信息
- 是否过度营销
- 标题是否和内容匹配
- 私信引导是否自然

## 适合放到什么工具

- Dify：做成一个“视频脚本生成器”
- n8n：串联素材收集、文案生成、飞书存稿
- Codex：基于本地资料和历史内容生成更稳定的稿件
