# CSV / Excel 数据分析报告工作流

## 场景

很多人把一份 CSV 丢给 AI，让它直接写分析结论。

风险很大。

因为 AI 可能：

- 没看清字段含义
- 忽略空值和异常值
- 把相关性说成因果
- 编造业务解释
- 没有给出可复核的计算过程

这个 workflow 的目标是：让 AI 帮你写报告，但每个关键结论都能追溯到数据和计算。

## Workflow

```text
CSV 文件
  -> 字段扫描
  -> 数据质量检查
  -> 指标计算
  -> 异常点识别
  -> 生成分析结论
  -> 生成复核表
  -> 人工确认业务解释
```

## AI 适合做什么

- 判断字段可能含义
- 生成分析框架
- 解释异常现象
- 组织报告语言

## 规则和代码必须做什么

- 行数、列数统计
- 空值比例
- 去重检查
- 指标计算
- 分组聚合
- 异常值识别
- 图表数据准备

## 输出要求

报告必须包含：

- 数据范围
- 字段说明
- 数据质量问题
- 核心指标
- 主要发现
- 异常点
- 可复核计算口径
- 后续建议

## 适合放到什么工具

- Codex：读取本地 CSV，跑脚本，生成报告
- Dify：做成上传文件分析应用
- n8n：定时读取数据文件并推送报告
- 飞书：自动写入分析文档
