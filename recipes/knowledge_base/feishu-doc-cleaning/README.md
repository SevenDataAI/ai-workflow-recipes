# 飞书知识库文章清洗工作流

## 场景

知识库最常见的问题不是内容少，而是内容看起来不像产品：

- 一整段挤在一起
- 标题层级混乱
- AI 味太重
- 没有图
- 没有案例
- 没有操作步骤
- 给用户看的内容里混入了内部话术

这个 workflow 的目标是把一篇文章从“草稿”变成“可售卖知识产品里的文章”。

## Workflow

```text
文章原文
  -> 结构扫描
  -> 删除内部提示
  -> 重排标题层级
  -> 补充真实场景
  -> 补操作步骤
  -> 生成流程图
  -> 去 AI 味润色
  -> 发布前检查
```

## AI 适合做什么

- 识别文章结构问题
- 重写表达
- 补充过渡句
- 生成流程图草稿
- 提取 FAQ

## 规则必须做什么

- 禁止出现“本页已做生产级重排”
- 禁止出现“用户会觉得值”
- 禁止出现“付费用户应该拿走什么”
- 禁止把销售话术写进交付文章
- 每篇文章至少有场景、步骤、案例、风险、交付物

## 发布前检查

一篇合格文章至少要回答：

1. 这个问题在真实工作里怎么发生？
2. 为什么普通做法不行？
3. 应该分几步做？
4. 每一步的输入和输出是什么？
5. 做错会有什么风险？
6. 最后能交付什么东西？

## 适合接入

- Codex：本地批量扫描和重写
- 飞书 CLI：写回知识库
- Dify：单篇文章清洗应用
- n8n：定时扫描待优化文章
