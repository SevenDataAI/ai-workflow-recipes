# 会议纪要到行动计划工作流

## 场景

很多会议纪要看起来很完整，但真正有用的信息不清楚：

- 谁要做什么
- 什么时候完成
- 风险在哪里
- 下次要汇报什么
- 哪些只是讨论，哪些已经形成决策

这个 workflow 的目标不是“总结会议”，而是把会议内容变成可执行计划。

## Workflow

```text
会议记录
  -> 提取背景
  -> 提取关键决策
  -> 提取待办
  -> 识别风险和阻塞
  -> 拆成行动项
  -> 生成汇报版摘要
  -> 人工确认
```

## AI 适合做什么

- 从长文本中提取结构化信息
- 区分讨论、决策、待办
- 生成给领导看的摘要
- 把口语化内容改成正式表达

## 规则必须控制什么

- 没有责任人的待办不能直接通过
- 没有时间点的计划要标记为待确认
- 不能把“讨论过”写成“已决策”
- 涉及承诺、排期、资源申请必须人工确认

## 输出格式

建议输出 4 块：

1. 本次会议结论
2. 后续行动计划
3. 风险和需要协调的问题
4. 给上级同步的话术

## 适合放到什么工具

- Dify：会议纪要分析应用
- n8n：录音转写完成后自动触发整理
- 飞书：自动写入会议文档和任务
- Codex：结合本地项目资料生成更具体的执行计划
