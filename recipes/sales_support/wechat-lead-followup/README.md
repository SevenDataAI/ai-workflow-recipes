# 微信私信线索跟进工作流

## 场景

用户从短视频、文章、GitHub 或知识库入口加你微信，第一句话通常很短：

- 求资料
- AI 数仓
- 能咨询吗
- 你这个适合我吗
- 完整版多少钱

如果每个人都手动回复，很容易出现两个问题：

1. 回复太快、太销售，用户有压力。
2. 回复太散，没有判断用户真实需求，后续转化断掉。

这个 workflow 的目标是：先判断用户意图，再给合适资料和下一步动作，而不是一上来就甩链接。

## Workflow

```text
用户消息
  -> 来源识别
  -> 意图分类
  -> 需求追问
  -> 资料匹配
  -> 回复生成
  -> 人工确认
  -> 线索记录
  -> 跟进提醒
```

## 哪些交给 AI

- 判断用户属于哪类需求
- 生成更自然的回复
- 根据用户回答推荐资料
- 总结用户画像和后续跟进重点

## 哪些必须规则化

- 不直接承诺效果
- 不一上来强推付费
- 没有确认方向前不发一堆资料
- 价格说明必须统一
- 付款入口必须人工确认后发送
- 重要用户要记录来源和关注点

## 用户意图分类

| 类型 | 典型表达 | 处理方式 |
| --- | --- | --- |
| 资料型 | 求资料、AI数仓、模板 | 发试读 + 询问关注方向 |
| 咨询型 | 可以咨询吗、想聊聊 | 先确认问题和背景 |
| 价格型 | 完整版多少钱 | 说明早鸟价和包含内容 |
| 犹豫型 | 我不太懂、适合我吗 | 降低门槛，给学习路径 |
| 企业型 | 公司想做、团队落地 | 引导到项目背景和具体问题 |

## 输出

- 用户意图
- 推荐回复
- 是否应该发试读
- 是否应该介绍完整版
- 是否需要后续提醒
- 线索记录摘要

## 适合接入

- n8n：触发、记录、提醒
- 飞书多维表格：线索沉淀
- Dify：生成回复草稿
- Codex：定期复盘聊天记录和转化问题
